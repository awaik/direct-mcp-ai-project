# Preflight Gate — проверка перед записью и запуском

Запуск кампании тратит деньги клиента и часто необратим. Перед любым write-tool сперва **попытайся кампанию опровергнуть**, потом ставь баллы. Система, которая оценивает собственный план, всегда завышает — закладывай это и придирайся сильнее.

## Шаг 1. Адверсариальный прогон (найди, чем кампания сольёт бюджет)

Пройди по плану с установкой «эту кампанию надо завернуть до открутки»:

- **Трекинг-пустышка.** Чем именно меряем результат? Для конверсий есть пиксель / `priced_event_type` / событие / CRM-передача? Нет — кампания крутит «в никуда». → стоп, чини трекинг.
- **Экономика наугад.** CPA/ДРР связаны с чеком и маржой или взяты с потолка? Цель «лиды любой ценой» без юнит-экономики → стоп.
- **Бюджет не на обучение.** Хватает на 5-10 целевых событий/день для автооптимизации? Если нет — либо честно называй это диагностическим тестом, либо меняй стратегию. Не обещай оптимизацию по конверсиям на голодном бюджете.
- **Посадочная/объект не держит.** Первый экран отвечает на обещание объявления? Форма/CTA/телефон работают? Редиректы без белого списка, попапы, фреймы, дорвейный контент? → риск; не делай вид, что реклама исправит посадочную.
- **Креатив-штамп / модерация.** Текст из «лучший / качественный / индивидуальный подход» без доказательства? Капс, цепочки восклицательных, эмодзи там, где VK запрещает? Нет блока «О компании»? → `references/creative-spec.md`.
- **Структура наугад.** 1 кампания = 1 цель = 1 объект? Не передроблено при мелком бюджете (алгоритму нужны данные)? Разные аудитории-гипотезы не смешаны в одну группу?
- **Выдуманные ID.** Каждый ID в плане есть в ID & facts ledger (фаза 3) с источником-инструментом? Пустой источник → значение не подтверждено → в план не идёт.

То, что нашёл, чини до выставления баллов.

## Шаг 2. Оси качества (floor-принцип)

Оцени каждую ось 0-10 **с конкретным фактом из плана** (без факта балл не засчитывается). Итог плана = **минимум по осям**, а не среднее. Anti-inflation: балл выше 9 — только с явным доказательством.

| Ось | Порог | Что проверяем |
|---|---|---|
| **Экономика** | 8 | CPA/ДРР/ROMI связаны с чеком, маржой, LTV — не «лиды любой ценой» |
| **Трекинг** | 9 (критично) | пиксель/цель/`priced_event_type`/событие/CRM реально считают результат; модель атрибуции задана |
| **Посадочная/объект** | 8 | первый экран держит оффер; форма/CTA/телефон работают; нет редиректов/попапов/дорвея |
| **Бюджет-на-обучение** | 8 | хватает на 5-10 событий/день для автооптимизации, иначе честно «диагностический тест» |
| **Креатив + оффер** | 8 | конкретика с цифрами, не штампы; модерация чистая; есть блок «О компании» (`creative-spec.md`) |
| **Аудитория/структура** | 8 | 1 кампания = 1 цель = 1 объект; гипотезы не передроблены и не смешаны |
| **Контракт записи** | 10 (бинарно) | план показан пользователю; все ID из ledger с источником; write-tools перечислены |

## Шаг 3. Решение

- **Слабейшая ось ≥ порога** → показывай план записи и жди текстового подтверждения (правило 2 в `SKILL.md`).
- **Любая ось < порога** → вернись в нужную фазу, почини **именно её**, прогони гейт заново. Не «подтяни балл», а реально исправь.

Floor работает буквально: план с осями [9,9,9,9,9,**5**] — это план на 5, а не на 8.3. Пятёрка тут — это, как правило, дыра в трекинге или бюджет, который не наберёт событий на обучение. И то и другое — стоп до запуска.

## После запуска гейт не выключается

На фазе 7 (оптимизация) те же оси работают как диагностика. Нет конверсий — спроси, **какая ось просела**: трекинг (событие не передаётся?), посадочная, оффер, аудитория, бюджет на обучение — а не «поднимем ставку». Слабейшая ось решает и здесь.
