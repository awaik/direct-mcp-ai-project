# Preflight Gate — проверка перед записью и запуском

Запуск поисковой кампании тратит деньги клиента и часто необратим. Перед любым write-tool сперва **попытайся кампанию опровергнуть**, потом ставь баллы. Система, которая оценивает собственный план, всегда завышает — закладывай это и придирайся сильнее.

## Шаг 1. Адверсариальный прогон

Пройди по плану с установкой «эту кампанию надо завернуть до открутки»:

- **РСЯ/wizard протёк.** `network_strategy` точно `SERVING_OFF` и нет `network_placement_network`? Нигде не всплыл Мастер кампаний / Мастер рекламных кампаний / Простой старт / скрытый wizard? Любой намёк → стоп.
- **Цель-агрегат.** В стратегии и в отчётах заданы явные `goals`? Goal ID реальные и бизнесовые (не «вовлечённые сессии» в одиночку)? Счётчик = домен посадочной? Без явных целей Conversions/CPA — агрегат по всему подряд, а не лиды или продажи. → стоп, чини трекинг.
- **Экономика наугад.** CPA/ДРР/ROAS связаны с чеком, маржой, CR лид→продажа — или взяты с потолка? «Конверсии любой ценой» без юнит-экономики → стоп.
- **Бюджет не на обучение.** Недельный бюджет набирает ≥10 целевых конверсий для конверсионной автостратегии? Если нет — либо честно называй это диагностическим тестом на `WB_MAXIMUM_CLICKS`, либо меняй стратегию. Не обещай оптимизацию по конверсиям на голодном бюджете.
- **Запросы-мусор.** Есть минус-фразы на уровне кампании и кросс-минусация между группами? Автотаргетинг не шире узкого контролируемого? Длинный хвост релевантен посадочной? Без минусовки конверсионная стратегия учится на мусоре.
- **Каша из интентов.** 1 группа = 1 интент? Бренд / конкуренты / горячий generic / тёплый / инфо не свалены в одну кампанию? Смесь интентов ломает и ставки, и отчётность.
- **Посадочная не держит.** Первый экран отвечает на запрос и обещание объявления? Оффер, цена/диапазон, CTA, телефон, форма работают на мобильном? Нет попапов, мешающих конверсии? → риск; не делай вид, что реклама исправит слабую посадочную.
- **Объявление-штамп.** Текст из «низкие цены / высокое качество / индивидуальный подход» без цифр? Заголовок не попадает в фразу группы? Меньше 2 вариантов на группу? → `references/ad-copy-spec.md`.
- **Выдуманные данные.** Каждый ID, частотность, ставка и бюджет в плане есть в ID & facts ledger (§7) с источником-инструментом? Пустой источник → значение не подтверждено → в план не идёт.

То, что нашёл, чини до выставления баллов.

## Шаг 2. Оси качества

Оцени каждую ось 0–10 **с конкретным фактом из плана** (без факта балл не засчитывается). Итог плана = **минимум по осям**, а не среднее. Anti-inflation: балл выше 9 — только с явным доказательством.

| Ось | Порог | Что проверяем |
|---|---|---|
| **Экономика** | 8 | CPA/ДРР/ROAS связаны с чеком, маржой, LTV, CR лид→продажа — не «конверсии любой ценой» |
| **Трекинг/цели** | 9 (критично) | явные `goals`, Goal ID реальные и бизнесовые, счётчик = домен, `attribution` задана |
| **Семантика/интент** | 8 | 1 группа = 1 интент; есть минусы и кросс-минусация; бренд/конкуренты/generic разделены |
| **Бюджет-на-обучение** | 8 | ≥10 целевых конверсий/нед для конверсионной автостратегии, иначе честно «диагностический тест» |
| **Посадочная** | 8 | первый экран держит запрос и оффер; форма/телефон/CTA работают; мобильная версия |
| **Объявления** | 8 | заголовок попадает в фразу; УТП с цифрами, не штампы; лимиты символов; ≥2 варианта (`ad-copy-spec.md`) |
| **РСЯ/wizard guard** | 10 (бинарно) | `network_strategy=SERVING_OFF`, нет network placement, нет Мастера кампаний/Простого старта |
| **Контракт записи** | 10 (бинарно) | план показан пользователю; все ID из ledger с источником; write-tools перечислены |

## Шаг 3. Решение

- **Слабейшая ось ≥ порога** → показывай план записи и жди текстового подтверждения.
- **Любая ось < порога** → вернись в нужную фазу, почини **именно её**, прогони гейт заново. Не «подтяни балл», а реально исправь.

Floor работает буквально: план с осями [9,9,9,9,9,5] — это план на 5, а не на 8.3. Пятёрка тут — это, как правило, дыра в трекинге (цели-агрегат), РСЯ, протёкшая в search-first, или бюджет, который не наберёт 10 конверсий на обучение. Всё это — стоп до запуска.

## После запуска

На фазе оптимизации (§8) те же оси работают как диагностика. Нет конверсий — спроси, **какая ось просела**: трекинг (цель не передаётся, отчёт без `goals`?), интент (мусорные запросы в `get_search_queries`?), посадочная, оффер, бюджет на обучение — а не «поднимем ставку». Слабейшая ось решает и здесь.
