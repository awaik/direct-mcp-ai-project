# SEO Site Core

Используй этот reference, когда пользователь просит SEO-ядро, ключи для сайта, полный список запросов, структуру категорий или контентные кластеры.

## Принцип

SEO/site-core оптимизируется не под максимум строк, а под полное покрытие релевантного спроса и понятную структуру страниц. Один кластер должен отвечать на вопрос: какую страницу нужно создать или улучшить под этот спрос?

## Матрица Покрытия Спроса

Для каждой категории/услуги проходи все слои:

| Слой | Что искать | Модификаторы |
|---|---|---|
| ВЧ / категория | общий спрос | категория, тип услуги, продукт |
| СЧ / характеристики | уточнения выбора | бренд, модель, материал, размер, назначение, цена, качество |
| НЧ / длинный хвост | точная потребность | для кого, для чего, проблема, действие, условия |
| Коммерческие | готовность купить | купить, заказать, цена, стоимость, доставка, рядом, под ключ |
| Вопросы | верхняя/средняя воронка | как выбрать, какой лучше, где купить, сколько стоит |
| Информационные | контент | что такое, для чего нужен, состав, инструкция, отзывы |
| Смежные | околоцелевой спрос | альтернативы, заменители, сопутствующие товары/услуги |
| Гео | локальный спрос | город, район, рядом, на дому, с доставкой |
| Бренды | брендовый спрос | свой бренд, товарные бренды, конкуренты отдельно |

## Правила Полноты

- Для широкой ниши нормальная цель — 200-300+ уникальных запросов, но только после удаления дублей и проверки интента.
- Для узкой ниши не растягивай ядро механически. Лучше меньше запросов, но с понятной страницей и интентом.
- Синонимы, разговорные варианты, транслит и опечатки добавляй только если они реально встречаются в Wordstat/поисковых запросах или явно типичны для ниши.
- Брендовые, конкурентные и навигационные запросы всегда выделяй отдельно.
- Смежный спрос не смешивай с коммерческими категориями: помечай как контент, РСЯ, ретаргетинг или тест.
- Информационные запросы не минусуй автоматически: для рекламы они часто мусор, для SEO это статьи, FAQ и верхняя воронка.

## Кластеризация Под Страницы

Разделяй кластеры, если:

- запросы требуют разных страниц;
- разные категории или услуги;
- разная аудитория;
- разные гео или условия доставки/выезда;
- разный этап воронки: купить, сравнить, узнать, решить проблему;
- выдача Яндекса показывает разные типы страниц.

Объединяй, если:

- интент совпадает;
- нужна одна посадочная;
- SERP похожий;
- текст страницы может естественно покрыть все фразы.

## Что Не Делать

- Не генерировать 200-300 фраз перестановками без Wordstat, SERP или понятной логики интента.
- Не смешивать "купить" и "как выбрать" в одну коммерческую страницу без отдельной контентной секции.
- Не смешивать конкурентов со своим брендом.
- Не выдавать SEO-задачу только в формате рекламных кампаний.
- Не добавлять опечатки, если нет признаков, что их реально ищут.
