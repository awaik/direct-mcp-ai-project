# Output Format

Финальный результат должен быть готовым к обсуждению и переносу в кампании, SEO-структуру сайта или контент-план.

Выбирай формат по режиму:

- `ppc/new_core/campaign_structure` — структура кампаний и групп.
- `seo_site_core` — структура страниц, категорий и контентных кластеров.
- `hybrid_core` — сначала разделение "Директ / SEO / контент / минус", затем детали.

## Краткое Резюме

```markdown
## Семантическое ядро: <ниша / проект>

Цель: <лиды / продажи / звонки>
Гео: <регионы>
Экономика: <CPA / ДРР / допущения>
Источник данных: <Wordstat / Директ / Метрика / выгрузки / SERP>

### Вывод
- Главные прибыльные кластеры: ...
- Риски: ...
- Что запускать первым: ...
```

## Таблица Кластеров

```markdown
| Кампания | Кластер | Интент | Температура | Посадочная | Приоритет | Комментарий |
|---|---|---|---|---|---|---|
| Поиск Generic | купить услугу | commercial | hot | /service | high | основной спрос |
```

## Ключи

```markdown
### Кампания: Поиск Generic

#### Группа: <кластер>
Посадочная: <url>
Интент: <commercial / transactional / informational / mixed>
Температура: <hot / warm / cold>
Приоритет: <high / medium / low>

Ключи:
- ...

Минус-слова группы:
- ...

Комментарий:
- почему кластер выделен отдельно;
- что проверить после запуска.
```

## Минус-Слова

```markdown
### Аккаунт
| Минус | Причина | Риск |
|---|---|---|

### Кампания
| Кампания | Минус | Причина | Риск |
|---|---|---|---|

### Группа
| Группа | Минус | Причина | Риск |
|---|---|---|---|
```

## План Запуска

```markdown
1. Запустить high-priority hot-кластеры.
2. Отдельно протестировать warm-кластеры с ограничением бюджета.
3. Бренд и конкурентов держать отдельно.
4. Через 7 дней проверить поисковые запросы, CPA и мусорный расход.
5. Через 14-21 день принять решения по расширению, минусам и посадочным.
```

## Если Данных Мало

Отдельно напиши:

- какие допущения приняты;
- какие данные нужны для уточнения;
- какие кластеры запускать тестом;
- какие кластеры не запускать до проверки экономики.

## SEO/Site-Core

Если пользователь просит семантическое ядро для сайта или полный список ключей, отдай не структуру рекламных кампаний, а структуру страниц.

```markdown
## SEO-семантика: <ниша / проект>

Гео: <регионы>
Аудитория: <сегменты>
Тип сайта: <интернет-магазин / услуги / SaaS / блог / лендинг>
Источник данных: <Wordstat / поисковые запросы / Метрика / SERP / допущения>

### Вывод
- Основные коммерческие разделы: ...
- Информационные кластеры: ...
- Брендовые и конкурентные блоки: ...
- Смежный спрос: ...
```

### Структура Страниц

```markdown
| Раздел / страница | Интент | Тип спроса | Приоритет | Комментарий |
|---|---|---|---|---|
| /category | commercial | category | high | основной спрос |
| /blog/how-to | informational | question | medium | контент для верхней воронки |
```

### Запросы По Категориям

Если нужен список, группируй запросы по категориям. Каждый запрос можно давать с новой строки или таблицей, если есть частотность.

```text
Категория: <название>
Интент: <commercial / informational / mixed / navigational>
Посадочная: <url или тип страницы>

<ключ 1>
<ключ 2>
<ключ 3>
...
```

Если есть частотность:

```markdown
| Категория | Запрос | Частотность | Интент | Страница |
|---|---|---:|---|---|
| <категория> | <ключ> | 1200 | commercial | /category |
```

### Обязательные Блоки Для SEO

- Коммерческие запросы: купить, заказать, цена, стоимость, доставка, гео.
- Информационные запросы: что такое, как выбрать, какой лучше, инструкция, сравнение.
- Вопросы и FAQ: отдельный блок для статей и FAQ-разметки.
- Брендовые запросы: свой бренд, товарные бренды, конкуренты отдельно.
- Смежный спрос: околоцелевые темы отдельно, не смешивать с основными категориями.
- Исключения: нерелевантные темы, мусор, юридически рискованные формулировки.

Если ядро большое, сначала покажи структуру категорий и 20-30 примеров, затем предложи выгрузить полный список в файл.
