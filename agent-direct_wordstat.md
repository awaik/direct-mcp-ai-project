# Яндекс Директ + Вордстат — Инструкции агента

## Философия

### Принципы работы
- **Фильтрация по умолчанию** — никогда не запрашивай всё подряд. Всегда ставь фильтры (states, field_names, campaign_ids), чтобы не получать мусор и не переполнять контекст.
- **Экономия токенов** — запрашивай только те поля, которые нужны для текущей задачи. Список кампаний — только Id и Name. Детальный анализ — все поля.
- **Верификация интента** — прежде чем выполнить действие (остановка, архивация, смена бюджета), убедись что правильно понял пользователя. Переспроси если есть сомнения.
- **Cache-first** — если данные уже получены в текущем разговоре, не запрашивай повторно. Используй то что есть.
- **Пошаговость** — сложные операции (создание кампании, массовые изменения) выполняй поэтапно с подтверждением пользователя на ключевых шагах.

### Красные флаги — остановись и спроси
- Пользователь просит удалить/архивировать больше 3 кампаний за раз
- Бюджет кажется неадекватно высоким или низким
- Ключевые фразы пересекаются с существующими кампаниями (каннибализация)
- Пользователь просит включить РСЯ или широкие запросы автотаргетинга

## Яндекс Директ — MCP-инструменты

MCP-сервер: `lidfly` (единый эндпоинт `https://lidfly.ru/mcp/v3`)

### Получение данных

#### Кампании — get_campaigns

**Правило: всегда фильтруй по states.**

| Запрос пользователя | Фильтр |
|---------------------|--------|
| "Покажи кампании" / "Мои кампании" | `states: ["ON"]` |
| "Покажи все включая остановленные" | `states: ["ON", "SUSPENDED"]` |
| "Покажи вообще всё" | без фильтра (только по явной просьбе) |
| "Какие остановлены?" | `states: ["SUSPENDED"]` |

**НЕ вызывай get_campaigns без фильтра states** — вернёт ВСЕ кампании (архивные, завершённые), результат бесполезен.

**НЕ ставь маленький limit** (5, 10) — лучше не указывать (по умолчанию 100).

**field_names — экономия токенов:**
- Обзор/список → `field_names: ["Id", "Name"]`
- Проверка статусов → `field_names: ["Id", "Name", "State", "Status"]`
- Детальный анализ → не указывай field_names (нужны все данные)

#### Порядок работы с API

1. **get_campaigns** — начинай с кампаний (с правильным фильтром!)
2. **get_adgroups** — группы объявлений (нужен campaign_id)
3. **get_ads** — объявления (фильтр по campaign_ids или adgroup_ids; для `RESPONSIVE_AD` запрашивай `responsive_ad_field_names` или используй `get_responsive_ads`)
4. **get_keywords** — ключевые фразы (фильтр по campaign_ids или adgroup_ids)
5. **get_campaign_stats** — статистика за период (campaign_id, date_from, date_to)
6. **get_search_queries** — реальные запросы пользователей (campaign_id, date_from, date_to)

**Важно**: всегда передавай campaign_ids / adgroup_ids, чтобы ответ не превысил лимит токенов.

### Управление кампаниями

| Действие | Инструмент | Условие |
|----------|-----------|---------|
| Остановить показы | **suspend_campaign** | Только для кампаний `ON` |
| Возобновить показы | **resume_campaign** | Только для кампаний `SUSPENDED` |
| Архивировать | **archive_campaign** | Сначала suspend, потом archive |
| Переименовать | **update_campaign** | Передать `name` |
| Изменить бюджет | **update_campaign** | Передать `daily_budget_amount` в рублях (целое число) |

**Цепочки:**
- Остановить: `suspend_campaign(campaign_id)`
- Возобновить: `resume_campaign(campaign_id)`
- Архивировать: `suspend_campaign(campaign_id)` → `archive_campaign(campaign_id)`

### Создание кампаний

#### Места показа — ВАЖНО

**Всегда только "Продвижение в поисковой выдаче":**
- Товарная галерея на поиске — ВЫКЛЮЧЕНА
- РСЯ (Рекламная сеть Яндекса) — ВЫКЛЮЧЕНА
- В API: `network_strategy: "SERVING_OFF"`

Не включать `NETWORK_DEFAULT`, `WB_MAXIMUM_CLICKS` для сетей без явной просьбы пользователя.

#### Автотаргетинг — категории запросов

**Только узкие и целевые:**
- Целевые запросы — ВКЛЮЧЕНЫ
- Узкие запросы — ВКЛЮЧЕНЫ
- Широкие запросы — ВЫКЛЮЧЕНЫ
- Сопутствующие запросы — ВЫКЛЮЧЕНЫ
- Альтернативные запросы — ВЫКЛЮЧЕНЫ

Не включать широкие/сопутствующие/альтернативные без явной просьбы.

#### Бюджет — РУБЛИ, НЕ микроюниты

**Бюджеты в Яндекс Директе НЕДЕЛЬНЫЕ, не дневные.**

**КРИТИЧЕСКИ ВАЖНО:** MCP-инструменты принимают бюджет в **РУБЛЯХ**. Передавай число как есть. **НИКОГДА не умножай на 1 000 000.**

- Параметр `daily_budget_amount` — это **недельный** бюджет
- 300 руб/нед → `daily_budget_amount: 300`, 1000 руб/нед → `daily_budget_amount: 1000`

Когда пользователь говорит "бюджет 400 рублей" — уточни: в неделю? Если да, передавай `400`.

#### Выбор стратегии — порядок запуска

WB_MAXIMUM_CONVERSION_RATE и AVERAGE_CPA — **два режима** стратегии "Максимум конверсий":

| Режим | Что делает | Когда |
|-------|-----------|-------|
| **WB_MAXIMUM_CONVERSION_RATE** | Максимум конверсий без потолка CPA | Старт, нет данных |
| **AVERAGE_CPA** | Конверсии с удержанием целевой цены | 10+ конверсий/нед стабильно |

**Правило: всегда запускай с WB_MAXIMUM_CONVERSION_RATE.**

Порядок:
1. Запуск → `WB_MAXIMUM_CONVERSION_RATE` + цель + недельный бюджет
2. Ждём 2-3 недели (10+ конверсий/нед)
3. Переключаем → `AVERAGE_CPA` с целевой ценой на основе реальных данных

**НЕ ставь AVERAGE_CPA на кампанию с 0 конверсий** — контрпродуктивно.

#### Порядок создания кампании

1. Проверь существующие кампании (каннибализация)
2. Проверь поисковый трафик ключевых фраз
3. Согласуй план с пользователем
4. Создай уточнения → `ad_extensions action:add`
5. Создай быстрые ссылки → `sitelinks action:add`
6. Создай кампанию → `add_unified_campaign`
7. Создай группы → `add_adgroup` / `add_adgroups` с `adgroup_type: "UNIFIED_AD_GROUP"`
8. Добавь ключевые фразы → `add_keywords` / `add_keywords_batch`
9. Добавь комбинаторные объявления → `add_responsive_ad`
10. Отправь на модерацию → `manage_ads action:moderate`

`add_campaign` → `add_adgroup` → `add_ad` / `add_ads` — только legacy/compatibility для старых текстовых кампаний. После 30.06.2026 даже TextAd creation-путь может вернуть фактический `RESPONSIVE_AD`; после создания проверяй ID через `get_ads` с `field_names: ["Id", "Type"]`.

#### Чеклист перед созданием

- Заголовки содержат ключевую фразу (лимиты: 56 / 30 / 81 символов)
- Тексты содержат призыв к действию (CTA)
- Для `RESPONSIVE_AD`: несколько разных заголовков и текстов внутри одного объявления, не варианты одного смысла
- Для legacy `TEXT_AD`: минимум 2-3 объявления на группу (A/B)
- Минус-фразы отсекают нецелевой трафик
- Нет каннибализации с существующими кампаниями
- Быстрые ссылки ведут на разные страницы
- UTM-метки настроены
- Бюджет адекватен
- Гео-таргетинг соответствует бизнесу

### Типовые сценарии и цепочки

**Анализ кампании:**
1. `get_campaigns` с campaign_ids → настройки, стратегия, бюджет
2. `get_adgroups` → `get_ads` с фактическим `Type` + `get_keywords`; для `RESPONSIVE_AD` — `get_responsive_ads`
3. `get_autotargeting` с adgroup_ids
4. `get_campaign_stats` за 30 дней
5. `get_search_queries` за 30 дней (если много — 14 или 7 дней)

**Проверка каннибализации:**
1. `get_campaigns` с `states: ["ON"]`, `field_names: ["Id", "Name"]`
2. Найти похожие по тематике
3. `get_keywords` для подозрительных кампаний
4. Сравнить с текущей кампанией

**Оптимизация ключевых фраз:**
1. `get_search_queries` — найти конверсионные (добавить как ключи) и нецелевые (в минус)
2. `get_criteria_stats` — найти фразы с высоким расходом без конверсий
3. `deduplicate_keywords` — очистить от дублей

## Яндекс Вордстат — MCP-инструменты

Вордстат входит в тот же единый MCP-сервер `lidfly` (эндпоинт `https://lidfly.ru/mcp/v3`) и работает через общий серверный Yandex Search API ключ LidFly. Отдельный Вордстат-токен, OAuth-подключение Директа, `connection_id` и `client_login` для `wordstat_*` не нужны и в arguments не передаются.

Все `wordstat_*` инструменты вызываются через `call_tool`, при необходимости сначала получи схему через `get_tool_schema`.

### Подбор фраз и частотность

**wordstat_top_requests** — основной инструмент для спроса и семантики. Возвращает точную частотность за последние 30 дней, похожие запросы и ассоциации. До 128 фраз за вызов инструмента; каждая фраза внутри считается отдельным внешним Search API запросом.

**check_search_volume** — не Вордстат. Используй его только как быструю проверку наличия трафика в Директе (есть/нет показов), а не как источник точной частотности.

Когда использовать `wordstat_top_requests`:
- Перед созданием кампании — оценить точный спрос и найти фразы-кандидаты
- При расширении семантики — получить ассоциации и похожие запросы
- При оптимизации — проверить потенциал новых фраз перед добавлением

### Тариф и лимиты

- Первые 20 биллинговых единиц Wordstat в месяц бесплатны.
- `wordstat_top_requests` списывает по числу фраз: 10 фраз = 10 единиц.
- `wordstat_dynamics` списывает 1 единицу.
- `wordstat_regions` списывает 2 единицы, потому что метод дороже в Search API.
- `wordstat_find_region` и `wordstat_user_info` бесплатны.
- Сверх пакета списание идёт с баланса. При нехватке баланса внешний API не вызывается.
- Неуспешные внешние запросы возвращаются автоматически: деньги на баланс, бесплатные единицы обратно в пакет.

### Инструменты

- `wordstat_top_requests` — частотность, ассоциации и идеи фраз
- `wordstat_dynamics` — сезонность и тренды по дням, неделям или месяцам
- `wordstat_regions` — спрос по регионам и индекс интереса
- `wordstat_find_region` — поиск ID региона для фильтров
- `wordstat_user_info` — доступность Search API, остаток бесплатного пакета и тариф

### Сезонность и тренды

`wordstat_dynamics` показывает динамику запросов с 2018 года для недель/месяцев и последние 60 дней для дневной детализации. Используй для:
- Определения сезонности бизнеса
- Планирования бюджетов (увеличивать в сезон, снижать вне сезона)
- Выявления растущих/падающих трендов

### Региональный спрос

`wordstat_regions` показывает спрос по регионам с индексом интереса. ID регионов бери через `wordstat_find_region`. Используй для:
- Определения приоритетных регионов для таргетинга
- Корректировки ставок по регионам
- Выбора регионов при создании кампании

**Правила работы с Вордстатом:**
- Проверяй ключевые фразы через `wordstat_top_requests` перед добавлением в кампанию, если нужен реальный спрос в цифрах.
- Отсеивай фразы с нулевым или нерелевантным спросом.
- Предупреждай о фразах с очень низким объёмом (< 10 запросов за 30 дней).
- Предлагай замены для слабых фраз на основе ассоциаций из `wordstat_top_requests`.
- Для больших списков сначала кластеризуй и убирай явный мусор, чтобы не тратить биллинговые единицы на заведомо нерелевантные фразы.

### Верификация интента — красные флаги

Перед добавлением фраз в кампанию проверь:

- **Информационный интент** — фразы типа "что такое", "как работает", "отзывы" → низкая конверсия, не добавлять в коммерческую кампанию без явной просьбы
- **Нецелевая семантика** — похожие слова с другим значением (омонимы)
- **Слишком широкие фразы** — высокая частотность, но размытый интент → высокий CPA
- **Брендовые запросы конкурентов** — проверь PROJECTS.md, показываемся ли мы по ним сознательно

### Автоматический подбор минус-слов

Вордстат помогает найти нецелевые ассоциации с ключевыми фразами. Используй для:
- Генерации списка минус-слов перед запуском кампании
- Расширения минус-слов для действующих кампаний

## Связка Директ + Вордстат

### Порядок: спрос → ключи → кампания → статистика

1. **Исследование спроса** (Вордстат)
   - Подбор фраз по тематике
   - Проверка частотности
   - Анализ сезонности
   - Определение регионов с максимальным спросом

2. **Подготовка семантики** (Вордстат + Директ)
   - Фильтрация фраз по интенту (коммерческие vs информационные)
   - Дедупликация с существующими кампаниями
   - Группировка по смысловым кластерам
   - Подбор минус-слов

3. **Создание кампании** (Директ)
   - Структура: кампания → группы → объявления + ключи
   - Настройки: бюджет, стратегия, регионы, автотаргетинг

4. **Оптимизация** (Директ + Вордстат)
   - Анализ поисковых запросов → новые ключи / минус-фразы
   - Проверка новых фраз через Вордстат перед добавлением
   - Корректировка бюджетов по сезонности

### Как результаты Вордстата превращаются в ключи кампании

```
Вордстат: "купить диван москва" — 12 000 показов/мес
  ↓ проверка интента (коммерческий ✓)
  ↓ проверка каннибализации (нет пересечений ✓)
  ↓ добавление в группу "Горячие — диваны"
  ↓ add_keywords в Директе
```

```
Вордстат: "диван своими руками" — 8 000 показов/мес
  ↓ проверка интента (информационный ✗)
  ↓ добавление "своими руками" в минус-фразы
```
